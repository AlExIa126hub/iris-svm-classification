Machine Learning project exploring Support Vector Machines (SVM) for classification using the Iris dataset.

# Project Overview

The goal of this project is to apply **Support Vector Machines (SVM)** to classify flowers in the Iris dataset and analyse how different kernel functions affect the classification results.

Support Vector Machines are powerful supervised learning algorithms that aim to find the optimal separating hyperplane between classes.

The workflow includes:

1. Loading and exploring the Iris dataset  
2. Selecting two classes for binary classification  
3. Training SVM models using different kernel functions  
4. Visualising the resulting decision boundaries  
5. Evaluating classification performance  
6. Extending the model to multiclass classification  

---

# Dataset

The Iris dataset contains measurements of iris flowers belonging to three species:

- Setosa  
- Versicolor  
- Virginica  

Each observation includes four numerical features:

- sepal length  
- sepal width  
- petal length  
- petal width  

The objective of the model is to classify each sample into the correct species based on these measurements.

---

# Support Vector Machines

Support Vector Machines are classification algorithms that attempt to find the **optimal hyperplane** separating different classes.

The optimal hyperplane is chosen to maximise the **margin**, which is the distance between the hyperplane and the closest data points from each class (called support vectors).

---

# Kernel Functions

Kernel functions allow SVMs to perform classification in higher-dimensional spaces when the data is not linearly separable.

The following kernels were tested:

### Linear Kernel

The linear kernel attempts to separate the data using a straight decision boundary.

This kernel works well when the data is approximately linearly separable.

---

### Polynomial Kernel

The polynomial kernel maps the data into a higher-dimensional space where nonlinear relationships between variables can be captured.

---

### Radial Basis Function (RBF) Kernel

The RBF kernel is one of the most commonly used kernels for SVMs. It can model complex nonlinear decision boundaries by measuring similarity between data points.

---

# Visualisation

The project visualises the **decision boundaries** generated by each kernel.

Plots show:

- the data points in feature space
- the separating hyperplane
- how the boundary changes depending on the kernel type

These visualisations help illustrate how different kernels handle nonlinear class separation.

---

# Multiclass Classification

SVM is inherently a **binary classifier**, but it can be extended to multiclass problems using different strategies.

Two common approaches are:

### One-vs-Rest (OvR)

A separate classifier is trained for each class against all other classes.

For a dataset with *n* classes, **n classifiers** are trained.

---

### One-vs-One (OvO)

A classifier is trained for every pair of classes.

For *n* classes, **n(n-1)/2 classifiers** are trained.

The final prediction is determined by a voting mechanism.

---

# Results and Analysis

The experiments demonstrate that:

- different kernel functions produce different decision boundaries
- nonlinear kernels can better capture complex class structures
- SVM can be successfully extended from binary classification to multiclass classification using OvR or OvO strategies
